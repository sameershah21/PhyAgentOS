# SO-101 Profile

This file describes the SO-101 6-DoF arm capability for the Critic to verify proposed actions against.

## Identity

- **Name**: SO-101
- **Type**: 6-DoF robot arm (open-source LeRobot-compatible)
- **Driver/Profile**: `so101`

## Degrees of Freedom

| Joint | Range (rad) | Description |
|---|---|---|
| j1 | -3.14 to 3.14 | shoulder yaw |
| j2 | -1.57 to 1.57 | shoulder pitch |
| j3 | -2.10 to 2.10 | elbow |
| j4 | -3.14 to 3.14 | wrist roll |
| j5 | -1.57 to 1.57 | wrist pitch |
| j6 | -3.14 to 3.14 | gripper rotation |

## Sensors

- **Joint encoders**: per-motor position feedback
- **Camera**: optional wrist-mounted (driver-dependent)

## Supported Actions

| Action | Parameters | Description |
|---|---|---|
| `home` | (none) | Move to default home pose |
| `move_to_pose` | `pose: [x, y, z]` (world coords, metres) | Move end-effector to a Cartesian pose |
| `grasp` | `target_id: str`, `pose: [x, y, z]` (optional, falls back to scene lookup) | Move to target and close the gripper on it |
| `release` | (none) | Open gripper, drop the held object |
| `gripper_open` | (none) | Open gripper (rejected if currently holding) |
| `gripper_close` | (none) | Close gripper |

## Physical Constraints

- **Reach envelope**: 0.50 m radius from shoulder
- **Shoulder offset (local)**: (0.0, 0.0, 0.35) above base mount point
- **Max payload**: 0.5 kg
- **Speed limits**: enforced by motor controller
- **Collision policy**: software-side reach check; no built-in collision avoidance — caller (Critic) must verify against scene

## Connection

- **Transport**: serial / USB (Feetech servos)
- **Default port**: `/dev/ttyUSB0`
- **Auth**: none
- **Reconnect Policy**: manual on driver init

## Runtime Protocol

- **Arm state channel**: `robots.<robot_id>.arm` in `ENVIRONMENT.md`
  - `joint_angles_rad: [j1..j6]`
  - `gripper_state: "open" | "closed"`
  - `holding: <object_id> | null`
  - `end_effector_world: [x, y, z]`
  - `stamp: <iso8601>`

## Authoring Rules

- This profile lists capabilities only. Live state belongs in `ENVIRONMENT.md`.
- The Critic uses the `Supported Actions` table to reject actions with the wrong shape, and the `Physical Constraints` section to reject out-of-envelope poses or overweight grasps.
