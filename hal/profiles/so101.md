# SO-101 Profile

This file describes the SO-101 6-DoF arm capability for the Critic to verify proposed actions against.

## Identity

- **Name**: SO-101
- **Type**: 6-DoF robot arm (open-source LeRobot-compatible)
- **Driver/Profile**: `so101`

## Degrees of Freedom

| Joint | Range (rad) | Description |
|---|---|---|
| j1 | -3.14 to 3.14 | shoulder yaw |
| j2 | -1.57 to 1.57 | shoulder pitch |
| j3 | -2.10 to 2.10 | elbow |
| j4 | -3.14 to 3.14 | wrist roll |
| j5 | -1.57 to 1.57 | wrist pitch |
| j6 | -3.14 to 3.14 | gripper rotation |

## Sensors

- **Joint encoders**: per-motor position feedback
- **Camera**: optional wrist-mounted (driver-dependent)

## Supported Actions

| Action | Parameters | Mock | Real HW | Description |
|---|---|---|---|---|
| `home` | (none) | ✅ | ✅ | Move to default home pose |
| `move_to_joints` | `joints: [j1..j6]` (radians) | ✅ | ✅ | Direct joint-space command |
| `move_to_pose` | `pose: [x, y, z]` (world coords, metres) | ✅ | ❌ | Mock-only Cartesian command (no IK on real hardware) |
| `grasp` | `target_id: str`, `pose: [x, y, z]` (optional, falls back to scene lookup) | ✅ | ❌ | Mock-only composite (move + close) |
| `release` | (none) | ✅ | ❌ | Mock-only (open gripper + drop tracking) |
| `gripper_open` | (none) | ✅ | ❌ | Mock-only (rejected if currently holding) |
| `gripper_close` | (none) | ✅ | ❌ | Mock-only |

On real hardware, the only movement primitives are `home` and `move_to_joints`. Higher-level actions return an error directing the caller to compose them from joint-space commands.

## Physical Constraints

- **Reach envelope**: 0.50 m radius from shoulder
- **Shoulder offset (local)**: (0.0, 0.0, 0.35) above base mount point
- **Max payload**: 0.5 kg
- **Speed limits**: enforced by motor controller
- **Collision policy**: software-side reach check; no built-in collision avoidance — caller (Critic) must verify against scene

## Connection

- **Transport**: serial / USB (Feetech STS3215 servos via lerobot's `FeetechMotorsBus`)
- **Default port**: `/dev/ttyUSB0` (override per platform: macOS uses `/dev/tty.usbmodem...`)
- **Calibration**: loaded at connect time from `~/.cache/huggingface/lerobot/calibration/robots/so101_follower/so101_follower.json` (written by `solo robo --calibrate follower` or `lerobot-calibrate`)
- **Auth**: none
- **Reconnect Policy**: manual on driver init; `close()` disconnects cleanly
- **Optional dep**: real-hardware mode requires `pip install -e '.[so101]'` (pulls `lerobot[feetech]`); mock mode has no extra deps
- **Port exclusivity**: cannot share `/dev/tty.usbmodem...` with `solo-cli` — close one before running the other

## Runtime Protocol

- **Arm state channel**: `robots.<robot_id>.arm` in `ENVIRONMENT.md`
  - `joint_angles_rad: [j1..j6]`
  - `gripper_state: "open" | "closed"`
  - `holding: <object_id> | null`
  - `end_effector_world: [x, y, z]`
  - `stamp: <iso8601>`

## Authoring Rules

- This profile lists capabilities only. Live state belongs in `ENVIRONMENT.md`.
- The Critic uses the `Supported Actions` table to reject actions with the wrong shape, and the `Physical Constraints` section to reject out-of-envelope poses or overweight grasps.
